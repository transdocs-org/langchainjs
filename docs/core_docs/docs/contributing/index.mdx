# 欢迎贡献者

你好！感谢你对参与 LangChain 的贡献感兴趣。  
作为一个处于快速发展领域的开源项目，我们非常欢迎各种形式的贡献，无论是新增功能、改进基础设施、优化文档还是修复 bug。

## 🗺️ 贡献指南

### 👩‍💻 贡献方式

你可以通过多种方式为 LangChain 做出贡献，以下是一些常见的贡献方式：

- [**文档**](/docs/contributing/documentation/style_guide)：帮助我们改进文档，包括本篇文档！
- [**代码**](./code.mdx)：帮助我们编写代码、修复 bug 或改进基础设施。
- [**集成**](./integrations.mdx)：帮助我们与你喜欢的供应商和工具进行集成。

### 🚩 GitHub Issues

我们的 [issues 页面](https://github.com/langchain-ai/langchainjs/issues) 会持续更新 bug、改进和功能请求。

我们使用了一套标签分类体系，以便对 issue 进行归类和查找。请使用这些标签来帮助组织 issue。

如果你开始处理某个 issue，请将其分配给自己。

如果你提交一个 issue，请尽量保持其聚焦于一个单一、模块化的 bug/改进/功能。
如果两个 issue 相关或互相依赖，请链接它们，而不要合并它们。

我们会尽量保持这些 issue 的时效性，但由于该领域发展迅速，有些 issue 可能会过时。
如果你发现这种情况，请告诉我们。

### 💭 LangChain 论坛

我们有一个 [论坛](https://forum.langchain.com/)，用户可以在上面提问、讨论设计决策以及提出新功能建议。

如果你有能力帮助回答问题，请积极参与！这将使维护者能够将更多时间投入到开发和 bug 修复上。

### 🙋 获取帮助

我们的目标是提供最简单的开发者环境。如果你在搭建环境时遇到任何困难，请务必联系维护者！我们不仅希望帮你解决问题，还希望确保未来贡献者的流程顺畅。

类似地，我们在代码库中执行了一定的代码规范、格式化和文档标准。
如果你发现这些标准难以遵守（甚至只是觉得烦人），请随时联系维护者寻求帮助——我们不希望这些阻碍优质代码进入代码库。

# 🌟 荣誉表彰

如果你的贡献被合并到某个发布版本中，我们会在 Twitter 上对你表示感谢（当然，仅在你愿意的情况下）！
如果你有希望我们提及的 Twitter 账号，请在 PR 中告知我们，或通过其他方式联系我们。
